# OriginFlow 手环协议测试工具

本工具包含两个Python脚本，用于测试OriginFlow手环的串口通信协议，并处理串口数据粘包问题。

## 文件说明

- `serial_protocol_test.py` - 完整的串口通信测试工具
- `protocol_test.py` - 协议解析测试脚本（无需硬件）
- `bluetooth_protocol_test.py` - 蓝牙GATT通信测试工具
- `bluetooth_protocol_test_simulation.py` - 蓝牙GATT协议测试脚本（无需硬件）
- `requirements.txt` - Python依赖包

## 协议格式

根据 `components/protocol_parser/protocol.h` 分析，协议格式如下：

```
| 字段        | 长度 | 说明                     |
|-------------|------|--------------------------|
| 帧头        | 1    | 0xAA                     |
| 协议版本    | 4    | "V001"                   |
| 数据包长度  | 2    | Little Endian            |
| 消息ID      | 4    | Little Endian            |
| 命令        | 2    | Little Endian            |
| CRC16       | 2    | Modbus CRC, Little Endian|
| 数据        | 可变 |                          |
```

固定包头长度：17字节

## 蓝牙GATT服务信息

根据 `main/bluetooth/gatts_nimble.c` 分析，设备支持蓝牙GATT服务：

- **服务UUID**: `FB349B5F-8000-0080-0010-0000FF0001`
- **特征UUID**: `FB349B5F-8000-0080-0010-0000FF0002`
- **支持操作**: NOTIFY, WRITE
- **蓝牙栈**: NimBLE

## 粘包处理机制

### 问题场景
1. **串口粘包**：
   - 多个数据包连续到达
   - 数据包分片传输
   - 无效数据干扰
   - 不完整数据包

2. **蓝牙GATT粘包**：
   - MTU限制导致的数据分片
   - 多个数据包在一次通知中
   - GATT通知丢失和重复
   - 无效数据干扰

### 解决方案
1. **帧头识别**：查找0xAA帧头
2. **长度验证**：读取长度字段验证完整性
3. **CRC校验**：使用Modbus CRC16验证数据完整性
4. **缓冲管理**：维护接收缓冲区，处理不完整数据
5. **数据清理**：丢弃无效数据，保证同步

## 常用命令

| 命令码 | 名称 | 说明 |
|--------|------|------|
| 0x0000 | 响应命令 | 设备响应 |

| 0x00FF | 重启命令 | 重启设备 |
| 0x0171 | 软件版本命令 | 获取软件版本 |

## 代码结构

### ProtocolParser类
- `crc16_modbus()` - CRC16计算
- `parse_packet()` - 解析单个数据包
- `process_data()` - 处理接收数据，解决粘包

### SerialProtocolTester类
- `connect()` - 连接串口
- `start_reading()` - 开始读取数据
- `send_test_packet()` - 发送测试数据包

## 注意事项

1. **波特率设置**：确保与设备波特率匹配（默认115200）
2. **串口权限**：Linux系统可能需要添加用户到dialout组
3. **数据格式**：所有数据使用Little Endian格式
4. **CRC校验**：使用Modbus CRC16算法
5. **粘包处理**：工具会自动处理各种粘包情况

## 故障排除

### 串口连接问题
- **无法连接串口**：
  - 检查串口设备路径
  - 确认串口权限
  - 检查波特率设置

- **数据解析失败**：
  - 确认设备使用正确的协议格式
  - 检查CRC校验是否正确
  - 验证数据包长度字段

### 蓝牙连接问题
- **无法扫描到设备**：
  - 确保设备蓝牙已开启并处于广播状态
  - 检查设备名称是否包含"OriginFlow"
  - 尝试靠近设备重新扫描

- **连接失败**：
  - 确认设备未被其他设备连接
  - 检查蓝牙权限（Linux可能需要蓝牙服务权限）
  - 尝试重启蓝牙服务

- **GATT服务找不到**：
  - 确认设备固件支持GATT服务
  - 检查服务UUID是否正确
  - 验证特征UUID是否匹配

- **通知接收不到数据**：
  - 确认通知已正确启用
  - 检查特征是否支持NOTIFY属性
  - 验证设备是否主动发送数据

### 粘包处理问题
- **工具已内置粘包处理逻辑**
- 查看缓冲区状态了解处理过程
- 检查帧头同步是否正常
- 验证CRC校验算法是否正确
